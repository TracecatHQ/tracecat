## `core.sql.execute_query`

Execute a parameterized SQL query on an external database.

### Inputs

<ResponseField name="query" type="string">

Required.

SQL query to execute. Use :param_name syntax for bound parameters. Do NOT use Tracecat expressions in the query string.

</ResponseField>

<ResponseField name="bound_params" type="object | null">

Optional.

Bound query parameters as a dictionary (injected with :param_name syntax). Supply dynamic values here, NOT within the query string. This is required for safe, parameterized SQL queries.

Default: `null`.

</ResponseField>

<ResponseField name="fetch_one" type="boolean">

Optional.

Return a single row instead of a list of rows. Defaults to False, which fetches all rows.

Default: `false`.

</ResponseField>

<ResponseField name="max_rows" type="integer">

Optional.

Maximum number of rows to return. Default 200. Prevents accidentally returning huge result sets.

Default: `200`.

</ResponseField>

### Examples

**Query an external database**

```yaml
- ref: recent_logins
  action: core.sql.execute_query
  args:
    query: |
      SELECT user_email, last_login_at
      FROM user_logins
      WHERE last_login_at >= :cutoff
      ORDER BY last_login_at DESC
    bound_params:
      cutoff: ${{ TRIGGER.cutoff }}
    max_rows: 100
```

Execute SQL in an in-process DuckDB engine

SQL to execute in an in-process DuckDB connection.

</ResponseField>

### Examples

**Run in-process SQL**

```yaml
- ref: aggregate_results
  action: core.duckdb.execute_sql
  args:
    sql: |
      WITH findings(id, severity) AS (
        VALUES (1, 'high'), (2, 'low'), (3, 'high')
      )
      SELECT severity, count(*) AS count

      GROUP BY severity
```
