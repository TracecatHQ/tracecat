## Overview

Agents can reference secrets and variables using Tracecat expression syntax.
Include `${{ SECRETS.<name>.<key> }}` or `${{ VARS.<name>.<key> }}` in the agent's `instructions` or `user_prompt` so the agent passes them as tool arguments.

<Note>
**Security model**: Agent tool calls go through a secure server-side proxy.
The LLM only ever sees the raw expression placeholder (e.g. `${{ SECRETS.slack.BOT_TOKEN }}`) — actual secret values are injected at the tool execution layer after the model responds, so the model never has access to your credentials.
</Note>

## Secrets

Use `${{ SECRETS.<secret_name>.<key> }}` to reference a secret stored in your workspace credentials.

<Prompt
  description="Sample: pass secrets to Slack and Jira tools"
  icon="lock"
  actions={["copy"]}
>
You are a security triage agent.

When you call Slack tools, pass `${{ SECRETS.slack.BOT_TOKEN }}` as the token.
When you call Jira tools, pass `${{ SECRETS.jira.API_TOKEN }}` as the API token and `${{ SECRETS.jira.EMAIL }}` as the user email.

Do not print or return secret values in your final answer.
</Prompt>

In a workflow action this looks like:

```yaml
- ref: triage_alert
  action: ai.agent
  args:
    model_name: gpt-4.1
    model_provider: openai
    instructions: |
      You are a security triage agent.

      When you call Slack tools, pass `${{ SECRETS.slack.BOT_TOKEN }}` as the token.
      When you call Jira tools, pass `${{ SECRETS.jira.API_TOKEN }}` as the API token
      and `${{ SECRETS.jira.EMAIL }}` as the user email.

      Do not print or return secret values in your final answer.
    user_prompt: |
      Investigate this alert and post a summary to Slack:

      ${{ TRIGGER.alert }}
    actions:
      - tools.slack.post_message
      - tools.jira.create_issue
```

## Variables

Use `${{ VARS.<variable_name>.<key> }}` for non-sensitive configuration such as base URLs, project IDs, or channel names.

<Prompt
  description="Sample: pass variables for Jira and Slack config"
  icon="variable"
  actions={["copy"]}
>
You are a security triage agent.

When you create Jira issues, use project key `${{ VARS.jira.project_key }}` and base URL `${{ VARS.jira.base_url }}`.
When you post to Slack, use channel `${{ VARS.slack.triage_channel }}`.
</Prompt>

In a workflow action:

```yaml
- ref: triage_alert
  action: ai.agent
  args:
    model_name: gpt-4.1
    model_provider: openai
    instructions: |
      You are a security triage agent.

      When you create Jira issues, use project key `${{ VARS.jira.project_key }}`
      and base URL `${{ VARS.jira.base_url }}`.
      When you post to Slack, use channel `${{ VARS.slack.triage_channel }}`.
    user_prompt: |
      Investigate this alert and file a Jira ticket:

      ${{ TRIGGER.alert }}
    actions:
      - tools.jira.create_issue
      - tools.slack.post_message
```

## Combining secrets and variables

Use secrets for credentials and variables for everything else.

<Prompt
  description="Sample: combined secrets and variables for alert triage"
  icon="sparkles"
  actions={["copy"]}
>
You are a security triage agent. Investigate the alert, enrich it, and take action.

Credentials:
- Slack token: `${{ SECRETS.slack.BOT_TOKEN }}`
- Jira API token: `${{ SECRETS.jira.API_TOKEN }}`
- Jira user email: `${{ SECRETS.jira.EMAIL }}`

Configuration:
- Jira project key: `${{ VARS.jira.project_key }}`
- Jira base URL: `${{ VARS.jira.base_url }}`
- Slack channel: `${{ VARS.slack.triage_channel }}`

Steps:
1. Look up the source IP and user in our enrichment tools.
2. Create a Jira issue with your findings.
3. Post a summary to the Slack triage channel.

Do not print or return secret values in your final answer.
</Prompt>

Full workflow example:

```yaml
- ref: investigate_and_respond
  action: ai.agent
  args:
    model_name: claude-sonnet-4-20250514
    model_provider: anthropic
    instructions: |
      You are a security triage agent. Investigate the alert, enrich it, and take action.

      Credentials:
      - Slack token: `${{ SECRETS.slack.BOT_TOKEN }}`
      - Jira API token: `${{ SECRETS.jira.API_TOKEN }}`
      - Jira user email: `${{ SECRETS.jira.EMAIL }}`

      Configuration:
      - Jira project key: `${{ VARS.jira.project_key }}`
      - Jira base URL: `${{ VARS.jira.base_url }}`
      - Slack channel: `${{ VARS.slack.triage_channel }}`

      Steps:
      1. Look up the source IP and user in our enrichment tools.
      2. Create a Jira issue with your findings.
      3. Post a summary to the Slack triage channel.

      Do not print or return secret values in your final answer.
    user_prompt: |
      Investigate this alert:

      ${{ TRIGGER.alert }}
    actions:
      - tools.jira.create_issue
      - tools.slack.post_message
      - tools.virustotal.get_ip_report
    max_tool_calls: 10
```

## Best practices

- **Never ask the agent to log or return secrets.** Add "Do not print or return secret values in your final answer" to your instructions.
- **Use variables for non-sensitive config.** Base URLs, project keys, and channel names belong in variables, not secrets.
- **One secret per integration.** Keep credentials grouped by service (e.g. a `jira` secret with `API_TOKEN` and `EMAIL` keys).
- **Use secret environments** to swap credentials across `prod`, `staging`, and `lab` without changing the prompt. See [secret environments](/automations/core-concepts/secrets#secret-environments).
