Watchtower records local agent sessions and tool calls made through Tracecat MCP.
You must be an organization administrator, and Watchtower must be enabled for your organization.

## Start collecting agent activity

<Steps>
  <Step title="Connect a local agent">
    Follow the [Tracecat MCP setup](/automations/tracecat-mcp#setup) for Codex,
    Claude Code, Cursor, or another supported MCP client.
  </Step>
  <Step title="Authenticate the connection">
    Complete OAuth sign-in or use a workspace-scoped personal access token.
  </Step>
  <Step title="Call a Tracecat tool">
    Start the MCP client and make a Tracecat tool call. Watchtower registers the
    client and session automatically when the connection initializes.
  </Step>
  <Step title="Open Watchtower">
    Open the workspace menu, select **Watchtower**, then open **Monitor**.

    <Frame>
      ![Watchtower Monitor with an agent, session, and tool calls selected](/img/audit-logs/watchtower-monitor.jpg)
    </Frame>
  </Step>
</Steps>

## Review activity

Use the filters above the monitor to narrow activity by agent status, session state, workspace, or tool-call status.
Select an agent to view its sessions, then select a session to inspect its tool calls.

Watchtower groups recognized clients as Claude Code, Codex, Cursor, Windsurf, OpenCode, or OpenClaw.
Clients that do not provide recognized metadata appear under `Unknown`.

| State | Meaning |
| --- | --- |
| Active agent | The agent was observed within the last 30 minutes. |
| Idle agent | The agent has not been observed for at least 30 minutes. |
| Blocked agent | An organization administrator disabled the agent. |
| Connected session | The MCP client initialized the session. |
| Awaiting initialize | OAuth completed, but the MCP client has not initialized the session. |
| Revoked session | An organization administrator revoked the session. |

The Monitor refreshes every five seconds.

## Control agent access

Click **Revoke** to block the selected session immediately.
Add an optional reason when you need the Monitor to retain context for the revoked state.

Click **Disable** to block every Tracecat MCP tool call from the selected local agent.
Select **Enable** when you want to allow the agent again.

## Data collection and retention

Watchtower stores:

- The MCP client type and reported client metadata.
- The attributed Tracecat user and, when available, workspace.
- Agent and session activity timestamps.
- Tool name, status, latency, and call timestamp.
- Argument names and structural metadata such as value type, length, and nested keys.
- A truncated error summary when a tool call fails.

Watchtower does not store raw tool argument values, prompts, or tool return values.
The Monitor displays agent, session, and tool-call activity from the last 30 days.
Tracecat prunes older records when Watchtower lists are queried, so records may
remain in storage until a later Monitor request triggers cleanup.

Watchtower covers activity sent through Tracecat MCP only.
It does not monitor unrelated local agent activity.
